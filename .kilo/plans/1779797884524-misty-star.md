# Plan: Fix laptop screen video reference

## Problem
The laptop 3D model in the AETHER section is configured to display a video at `/assets/videos/pc-screen.mp4`, but the actual file in the assets directory is named `pc video 2.mp4` (with a space). This mismatch prevents the video from loading and displaying on the laptop screen.

## Solution
Update the video source path in the laptop scene JavaScript code to reference the correct filename.

## Files to Modify
- `src/pages/index.astro` - Update the video element source path

## Changes Needed
In the laptop scene code (around line 760), change:
```javascript
videoElement.src = '/assets/videos/pc-screen.mp4';
```
to:
```javascript
videoElement.src = '/assets/videos/pc%20video%202.mp4';
```
or alternatively:
```javascript
videoElement.src = '/assets/videos/pc video 2.mp4';
```

## Implementation Steps
1. Locate the laptop scene video initialization code in `src/pages/index.astro`
2. Update the videoElement.src property to point to the correct file
3. Save the file
4. Restart development server if running locally to see changes

## Verification
After implementation, the laptop screen in the AETHER section should display the video content from `pc video 2.mp4` when the site is viewed.
